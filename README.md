# pyCycle Edu

This repository is an education/workspace wrapper around upstream pyCycle.
It keeps upstream pyCycle as a read-only submodule and stores teaching assets, reference sources, development logs, and future wrapper code outside `upstream/pyCycle`.

## Current Teaching Goal

Build a Traditional Chinese course that teaches students how to prompt AI to:

- manage a reproducible Python environment with `uv` and `.venv`,
- use pyCycle examples as read-only reference models,
- collect and cite public engine data in `Reference_sources/`,
- plan a PySide6 UI that approximates entry-level GasTurb-style workflows,
- generate Traditional Chinese and English engineering reports.

The first teaching case is a CFM56-7B-class high-bypass turbofan, using pyCycle's upstream `high_bypass_turbofan.py` example as the initial learning scaffold. The example is not treated as a calibrated CFM56-7B engine deck.

## Repository Layout

| Path | Purpose |
|---|---|
| `upstream/pyCycle/` | Read-only upstream pyCycle submodule. Do not edit or move the gitlink unless explicitly syncing upstream. |
| `Reference_sources/` | External references, source manifest, and source governance rules. |
| `docs/slides/` | Teaching slide generator, generated PPTX, and local visual assets. |
| `docs/dev-log/` | Required development logs for meaningful work. |
| `docs/research/` | Research goals and teaching/research progress notes. |
| `docs/sessions/` | Session summaries for handoff between humans and coding agents. |
| `scripts/` | Repository maintenance checks, including upstream submodule verification. |

## Generated Teaching Deck

The current course planning deck is:

```text
docs/slides/pycycle_ai_prompting_course.pptx
```

It is generated by:

```powershell
uv run --with python-pptx --with pillow python docs/slides/build_pycycle_ai_prompting_ppt.py
```

The deck uses a warm beige classroom-friendly visual style, larger text for projection accessibility, and embedded bitmap illustrations generated into `docs/slides/assets/`.

## Required Workflow

Before analysis or edits:

```powershell
npx gitnexus analyze --embeddings
```

Before committing:

```powershell
./scripts/verify-upstream-submodule.ps1
```

Meaningful work must be recorded under `docs/dev-log/`.

## Version / Release Notes

This repository currently has no package version file and no release tag policy. The current work is documentation and teaching-material preparation, so it should be committed and pushed but not published as a GitHub Release unless a version/tag policy is added later.
