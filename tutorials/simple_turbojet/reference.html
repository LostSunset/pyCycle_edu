1. 渦輪噴射引擎 101 / Turbojet Primer

渦輪噴射引擎是最早的實用噴射推進形式（1937 年 Whittle / von Ohain）。 它的理想熱力循環是 布雷頓循環（Brayton Cycle）： 等熵壓縮 → 等壓加熱 → 等熵膨脹 → 等壓散熱。

1.1 循環四步驟

步驟 / Step 過程 / Process 實機元件 / Real Component 物理意義 / Meaning
1 → 2 Isentropic compression
等熵壓縮		 Inlet + Compressor
進氣道 + 壓縮機		 提高空氣壓力與溫度，準備進入燃燒室；理想為等熵，實際有效率損失。
2 → 3 Isobaric heat addition
等壓加熱		 Combustor
燃燒室		 噴入燃油燃燒，溫度大幅升高；理想壓力不變，實際有壓力損失 dP/P。
3 → 4 Isentropic expansion
等熵膨脹		 Turbine
渦輪		 高溫氣體驅動渦輪、由軸帶動壓縮機；功率必須等於壓縮機所耗（軸功率平衡）。
4 → 5 → 0 Isobaric heat rejection
等壓散熱		 Nozzle
噴嘴 + 大氣		 剩餘高壓氣體經噴嘴加速排出，產生推力；之後在大氣中散熱回到環境狀態。

1.2 推力與比油耗

淨推力 / Net Thrust (Fn)：噴嘴出口動量 − 入口動量（ram drag）。 這是引擎實際「推飛機」的力量。

比油耗 / Thrust Specific Fuel Consumption (TSFC)：每單位推力每小時消耗 多少燃油，數值越小越省油。對民航發動機是核心競爭指標。

總壓比 / Overall Pressure Ratio (OPR)：燃燒室入口總壓 ÷ 大氣總壓。 OPR 越高熱效率越高，但會推升 T3、T4 與材料負擔。

1.3 為何需要 Newton solver？

引擎設計同時要滿足幾個彼此牽連的條件：

  • 軸功率平衡：渦輪輸出 = 壓縮機消耗
  • 推力 = 目標推力（Fn target）
  • T4 = 渦輪入口溫度上限

這些方程式都互相耦合（feedback loop），無法照單向順序逐步解出， 必須用迭代求解器（OpenMDAO 的 NewtonSolver）反覆收斂。 這正是 BalanceComp 與 n2.html 上三角連線的物理本質。

站位物理意義

站位pyCycle 名稱所在元件正體中文關鍵物理量
0fc.Fl_OFlightConditions 大氣 / 自由流 大氣 P, T（隨 alt 而變）。本模型 SLS 時 Pt=14.7 psia, Tt=518.7 degR。
1inlet.Fl_OInlet 進氣道出口 受 ram recovery 效應；本模型靜地 MN→0，Pt 略低於 0 站。
2comp.Fl_OCompressor 壓縮機出口 Pt 大幅上升（×13.5），Tt 上升至 ~1200 degR。
3burner.Fl_OCombustor 燃燒室出口 Tt = T4 ≈ 2370 degR，Pt 因燃燒損失下降 3%。

讀圖技巧：T-S 圖上看「站 3 → 4」斜率代表渦輪做功； P-h 圖上看「站 2 → 3」垂直幾乎不變代表等壓加熱（除了 3% 損失）； 站 0 → 1 在靜地時幾乎重疊（沒有 ram 效應）。

4. pyCycle 元件參考 / Component Reference

FlightConditions ─ 飛行條件

  • 輸入：alt（高度）、MN（馬赫數）
  • 輸出：大氣 P, T；建立 0 站位流動。
  • 實機對應：大氣模型 + Pitot 入口靜壓。

Inlet ─ 進氣道

  • 輸入：來自 fc.Fl_O 的流動 + MN（進氣道內馬赫數）
  • 內部：計算 ram drag F_ram。
  • 實機對應：飛機進氣道；超音速時有 shock recovery 損失。

Compressor ─ 壓縮機

  • 關鍵設計變數：PR（壓比）、eff（等熵效率）
  • 使用 map：pyc.AXI5（軸流式 5 級壓縮機通用 map）
  • 軸接口：輸出反扭矩 trq 到 shaft.trq_0。
  • 實機對應：多級軸流壓縮機，例如 J85 共 9 級。

Combustor ─ 燃燒室

  • 輸入：FAR（油氣比）由 BalanceComp 解出
  • 內部：計算 Wfuel，加熱氣體到 T4。
  • 損失：dPqP=0.03（3% 壓損）
  • 實機對應：環形或筒形燃燒室。

Turbine ─ 渦輪

  • 關鍵設計變數：PR（被 BalanceComp 解出）、eff
  • 使用 map：pyc.LPT2269（低壓渦輪通用 map）
  • 軸接口：輸出 trq 到 shaft.trq_1。
  • 實機對應：單級或多級反作用式渦輪。

Nozzle ─ 噴嘴

  • 型態：nozzType='CD'（Convergent-Divergent，收斂擴散）
  • 損失：Cv=0.99（99% 速度係數）
  • 約束：出口 Ps 必須匹配 fc.Fl_O:stat:P。
  • 實機對應：定型噴嘴（亞音速）或可變幾何（超音速作戰機）。

Shaft ─ 主軸

  • 連接：compressor.trq 與 turb.trq
  • 輸出：pwr_net（淨功率，設計點 = 0）
  • 本模型：單軸（num_ports=2），共用 8070 rpm。

Performance ─ 性能彙整

  • 計算：OPR、淨推力 Fn、TSFC
  • 輸入：perf.Pt2, perf.Pt3, perf.Wfuel_0, perf.ram_drag, perf.Fg_0
  • 是觀察輸出而非物理元件。

BalanceComp ─ 平衡元件 / Newton 變數容器

  • 設計點解 (W, FAR, turb_PR) 滿足 (Fn, T4, shaft.pwr_net=0)
  • 離設計點解 (W, FAR, Nmech) 滿足 (nozz.Throat:stat:area, Fn, shaft.pwr_net=0)
  • 每條方程式以 lhs:<name> 與 rhs:<name> 表示左右式。

5. GUI 輸入欄位逐項解釋 / GUI Input Reference

左側面板「Design Point Inputs / 設計點輸入」共 7 個欄位：

欄位變數路徑單位預設意義 / 調校建議
Altitude / 高度 DESIGN.fc.alt ft0 飛行高度。0 = 海平面 (SLS)。每升高 1000 ft，大氣 P 約降 3.5%。
Mach / 馬赫數 DESIGN.fc.MN —1×10⁻⁶ 近 0 即代表靜地。設計點 SLS 取極小值避免奇異。巡航典型 0.2–0.85。

T4 target / 渦輪入口溫度 DESIGN.balance.T4_target degR2370 渦輪材料的溫度極限。提高會增推力但減壽命。不可超過材料規格。
Compressor PR / 壓縮機壓比 DESIGN.comp.PR —13.5 壓縮機提高壓力的倍數。高 PR 提高熱效率但需要更多級。
Compressor η / 壓縮機效率 DESIGN.comp.eff —0.83 等熵效率。實機現代軸流壓縮機通常 0.83–0.89。

離設計點 OD0 / OD1 是寫死的：
  • OD0：MN≈0、alt=0、Fn target=11000 lbf（接近設計點但較低推力）

  • OD0.fc.alt / OD0.fc.MN / OD0.balance.Fn_target
  • OD1.fc.alt / OD1.fc.MN / OD1.balance.Fn_target

8. n2.html 詳解

8.1 它在回答什麼問題？

「這個模型內部資料怎麼流？哪裡是回授？Newton solver 在解什麼？」

8.2 介面元素

  • 對角線元件（灰色方塊）：依執行順序排列。
  • 下三角（藍/橘色塊）：前饋連線 — 上游送下游，自然順序可解。
  • 上三角（紅色塊）：回授連線 — 必須 Newton solver 才能解。
  • 左側樹：可展開的系統階層 (DESIGN / OD0 / OD1)。
  • 頂部 Search：高亮特定變數所有連線。

8.3 本模型的關鍵連線

下三角（前饋，藍色）

  • fc.Fl_O → inlet.Fl_I → comp.Fl_I → burner.Fl_I → turb.Fl_I → nozz.Fl_I（氣流接力）
  • comp.trq → shaft.trq_0、turb.trq → shaft.trq_1（軸扭矩）
  • perf.Fn → balance.lhs:W（推力進 balance 左式）
  • burner.Fl_O:tot:T → balance.lhs:FAR（T4 進 balance 左式）
  • shaft.pwr_net → balance.lhs:turb_PR（軸功率進 balance 左式）

上三角（回授，紅色）─ Newton solver 戰場

  • balance.W → inlet.Fl_I:stat:W
  • balance.FAR → burner.Fl_I:FAR
  • balance.turb_PR → turb.PR

看到三條上三角線就知道：「本案 Newton 在解 3 維方程式」。

8.4 DESIGN ↔ OD 連線

  • DESIGN.comp.s_*（map 縮放係數） → OD0.comp.s_*、OD1.comp.s_*
  • 同理 turb、nozz
  • DESIGN.nozz.Throat:stat:area → OD*.balance.rhs:W（噴嘴喉部鎖定）

視覺呈現了「設計尺寸定了 → OD 不能改尺寸，只能改操作點」的工程慣例。

9. 工程實例 / Engineering Scenarios

實例 A ─ 起飛推力規格驗證

需求：SLS 推力 11800 lbf，T4 不超過 2370 degR。

  1. GUI：alt=0, MN=1e-6, fn=11800, T4=2370
  2. 跑完看 summary，確認 Fn ≈ 11799.998 lbf。

  4. n2.html DESIGN 子區塊截圖貼進交付文件。

實例 B ─ 冷天 / 熱天靈敏度

本模型未直接暴露 dT_amb，但可以透過修改 solve_problem 加進去。 冷天 Tt 降低 → ρ 升高 → 同樣 W 對應更大壓縮功 → Fn 升高，TSFC 改善；熱天相反。 這是民航起飛限重最大的影響因子之一。

實例 C ─ 巡航油耗評估

需求：5000 ft / Mach 0.2 / 8000 lbf 巡航時的 TSFC。

  1. 查看 comparison.png 中 OD1 的 TSFC bar。
  2. 對比 OD0（地停）、DESIGN（SLS）三點 TSFC。

實例 D ─ 壓縮機選型 trade study

兩款候選壓縮機（PR 同為 13.5，eff 略異）。在 GUI 跑兩次，各自存 output/ 後 rename。 對比 compressor_map_DESIGN_*.png 上的工作點與 surge line 距離（surge margin）。

10. 常見錯誤排除 / Troubleshooting

10.1 Newton 不收斂

  • 症狀：log 顯示 NL: Newton Converged in N iterations 失敗， 或出現 RuntimeWarning: invalid value encountered in sqrt 後 NaN。
  • 檢查：
    1. n2.html 上三角是否有預期外的連線？多餘的回授會增加 Newton 維度。
    2. inputs.html 過濾 balance，確認 LHS / RHS 對應的物理量合理（單位、量級）。
    3. 初始猜值 prob['DESIGN.balance.FAR'] = 0.0175... 是否過於偏離？
    4. 輸入 T4 是否高到讓燃燒室不可能達到（或低到 FAR 變負）？

10.2 PySide6 報錯 ModuleNotFoundError

uv pip install PySide6

10.3 圖表中文亂碼

GUI 啟動時會嘗試 Microsoft YaHei → SimHei → Arial Unicode MS → DejaVu Sans 字體。 若仍亂碼，安裝 Noto Sans TC 字體，並在 simple_turbojet.py 開頭調整 matplotlib.rcParams['font.sans-serif']。

10.4 output/ 沒有出現 / 看不到舊版的 simple_turbojet_out

v0.2.0 後所有輸出都集中在 tutorials/simple_turbojet/output/。 如果在 repo 根目錄看到舊的 simple_turbojet_out/，那是早期版本的殘留，可手動刪除。

10.5 OpenMDAO set_reports_dir has been removed

OpenMDAO 3.40 移除了該函式。本模組改採 os.chdir(OUT_DIR) + _flatten_openmdao_reports() 把 reports 搬到 OUT_DIR。如未來 3.41+ 提供官方 redirect API，可改用之。

10.6 n2.html 打不開或卡頓

檔案 ~1 MB 含內嵌 JS，建議用 Chrome / Edge / Firefox 開啟。停用瀏覽器擴充 （特別是廣告封鎖）可改善載入速度。

10.7 站位標籤又重疊了

v0.2.0 已用 STATION_LABEL_OFFSETS 為每個站位安排固定方向。如果你大幅修改 了 PR、T4 等導致兩點極端靠近，可修改 simple_turbojet.py 內該常數的偏移向量。
